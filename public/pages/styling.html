Modern CSS

Modern web applications are built on top of heavy tooling for dealing with CSS, relying on lots of NPM packages and a bunch of build steps. A vanilla web application however has to choose a lighter weight path, abandoning the traditional modern CSS approaches and choosing strategies that are browser native.

Reset

Resetting the styles to a cross-browser common baseline is standard practice in web development, and vanilla web apps are no different.

Other options, in increasing order of complexity:

modern-normalize

A more comprehensive solution for resetting CSS for modern browsers.

Include it with:

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/modern-normalize@2.0.0/modern-normalize.min.css" />

Kraken

A starting point for front-end projects. It includes a CSS reset, typography, a grid, and other conveniences.

Include it with:

<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/Kraken/13.0.0/main.min.css" />

Tailwind

If you're going to be using Tailwind anyway, you may as well lean on its CSS reset.

Include it from CDN.

Fonts

Typography is the keystone of a web application. A lean approach like vanilla web development should be matched with a lean approach for typography. Modern Font Stacks describes a varied selection of commonly available fonts with good fallbacks, avoiding the need to load custom fonts.

This site uses the Geometric Humanist stack for normal text, and the Monospace Code stack for source code.

The Toolbox

In any real world web project the amount of CSS quickly becomes unwieldy unless it is well-structured. Let's look at the toolbox that CSS provides us in modern browsers to provide that structure.

@import

The most basic structuring technique is separating CSS into multiple files. We could add all those files in order as <link> tags into the index.html but this quickly becomes unworkable if we have multiple HTML pages. Instead it is better to import them into the index.css

Shadow DOM

Adding a shadow DOM to a web component allows us to further isolate its styles from the rest of the page. For example, the x-badge component from the previous page restyles all span elements inside its CSS, without affecting the containing page or the badge's child elements.

While all CSS files that need to apply to the shadow DOM must be imported into it explicitly, CSS variables are passed into the shadow DOM.

Know that should you want to use custom fonts inside a shadow DOM, they must first be loaded into the light DOM.

Structure

There are many ways to organize CSS files in a repository, but this is the one used here:

/index.css
The root CSS file that imports all the other ones using @import.
/styles/reset.css
The reset stylesheet is the first thing imported.
/styles/variables.css
All CSS variables are defined in this separate file, including the font system.
/styles/global.css
The global styles that apply across the web pages of the site.
/components/example-component/example-component.css
All styles that aren't global are specific to a component, in a CSS file located next to the component's JS file, and imported into index.css.
